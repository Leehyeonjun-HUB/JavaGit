
----------데이터베이스 
서로 관련성을 가진 데이터들을 데이터 간의 중복성을 최소화 해서 체계적으로 모아 놓은것

----------관계형데이터베이스
데이터를 테이블(표)로 표현
데이터를 항목으로 구분하여 테이블로 구성하는 것

----------관계형 데이터베이스 다운 개념의 구조

RDM는 내포관계를 허용X
거대 단일 테이블로 표현을 하면 중복이 발생
JOIN활용해야 함

----------ERD의 구성요소
개념적모델링 - Entity 
구체적인 데이터 - Attribute(속성)
연관성을 표현 - Relation 
행 - Tuple -> Row

그림표시- (개체 - 사각형, 속성 - 타원형, 키속성-이름하단밑줄표시)

식별자 - 밑줄이 그어지면 primary key (각각의 테이블에 유일무이한 식별자)

----------릴레이션쉽
-카디널리티
	1:1, 1:N, N:M 관계
-옵셔널리티
	옵션 (선택)
	다이어그램에서 동그라미(O)로 표시

**********데이터모델링단계************

-요구사항분석 ->개념적 데이터 모델링 -> 논리적 데이터 모델링 -> 물리적 데이터 모델링 -> 구현

----------업무파악 (요구 사항 분석)
- 데이터베이스의 용도 파악

결과물 : 요구 사항 명세서
	
----------개념적 데이터 모델링
- DBMS에 독립적인 개념적 구조 설계
- 개체와 속성 추출, 관계 추출, ER다이어그램작성

결과물 - 개념적 스키마(E-R 다이어그램)

----------논리적 데이터 모델링 
- 관계형 데이터베이스에 맞는 이상적인 표 작성
entitiy -> table
attribute -> column
relation -> PK,FK

5가지 릴레이션 변환규칙에 따라 개념적설계를 릴레이션 스키마로 변환
1. 모든 개체는 릴레이션으로 변환
2. 1:1관계는 외래키로 표현
3. 1:N관계는 외래키로 표현
4. N:M관계는 릴레이션으로 변환
5. 다중값 속성은 독립 릴레이션으로 변환

결과물 - 논리적 스키마(릴레이션 스키마)

-----------물리적 데이터 모델링
-이상적인 표를 구체적인 제품에 맞는 현실적인 표로 작성
- 필요한 인덱스 구조나 내부 저장 구조 등 물리적 구조설계

결과물 - 물리적 스키마

-----------구현
SQL문을 작성한 후 이를 DBMS에서 실행하여 데이터베이스 생성

-----------정규화
데이터의 삽입,수정,삭제,연산을 수행할 때 발생하는 Anomaly(이상현상)을 제거하면서 데이터베이스를 올바르게 설계해 나가는 과정
-이상현상이 발생하지않으려면 관련있는 속성들로만 릴레이션을 구성해야함

제1정규형(1NF) - 릴레이션에 속한 모든 속성의 도메인이 원자 값으로만 구성
제2정규형(2NF) - 릴레이션이 제1정규형에 속하고, 기본키가 아닌 모든 속성이 기본키에 완전 함수 종속     (부분함수적종속을 제거해야 제1정규형이 제2정규형을 만족)
제3정규형(3NF) - 릴레이션이 제2정규형에 속하고, 기본키가 아닌 모든 속성이 기본키에 이행적 함수 종속이 되지 않을 시
BCNF(보이스-코드 정규형) - 릴레이션에서 모든 결정자가 후보키가 되도록 하는 과정
제4정규형(4NF) - 다치종속 관계가 성립되는 경우 분해하는 과정
제5정규형(5NF) - 조인종속이 후보키를 통해서만 성립이 되도록 하는 정규형

-----------역정규화
정규화된 릴레이션을 다시 통합하거나 분할하여 구조를 재조정하는 과정
